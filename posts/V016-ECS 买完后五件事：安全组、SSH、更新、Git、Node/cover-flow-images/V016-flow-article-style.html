V016 浅色图文流程

ECS 买完后：先初始化，再装工具

保留本条关键流程：实例、安全组、SSH、用户权限、系统更新、Git / Node、快照与费用。
STEP 1

ECS / 安全组

确认实例运行、公网 IP、镜像、22 端口和可信来源。

STEP 2

SSH

确认用户名、密钥或密码、密钥文件权限；Workbench 只做备用判断。

STEP 3

用户权限

进入服务器后看 whoami 和 /etc/os-release，区分 root 与普通用户。

STEP 6

快照 / 费用

重要操作前考虑快照；测试机不用时看停止和释放提示。

STEP 5

安装工具

安装并验证 Git、Node、npm；输出版本才算基础可用。

STEP 4

更新环境

按镜像选择 apt、dnf 或 yum；具体命令需录屏前实测确认。

正常先查实例和安全组，再连 SSH。
提示不要长期开放所有来源。
验证git、node、npm 都要看版本输出。
结尾复盘

ECS 控制台 / 安全组 / 公网 IP / SSH / root / 普通用户 / 系统更新 / Git / Node / 快照
