# 阿里云 ECS 买完后怎么初始化：安全组、SSH、Git、Node 五步检查

这条讲阿里云 ECS 创建后最先做的初始化检查：确认实例状态和公网 IP，检查安全组 22 端口，用 SSH 或 Workbench 登录服务器，按镜像更新系统，安装并验证 Git、Node、npm，再补上快照和费用风险提醒。后续再装 Codex 或 Claude Code，会更容易排查问题。

这篇适合刚开始接触Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按这条顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

(1)常见卡点

控制台能看到实例，但不知道公网 IP 在哪里；SSH 连不上，不知道是安全组、用户名、密码/密钥、端口还是实例状态问题；安全组规则随手放开 `0.0.0.0/0`，但没有理解风险；root 和普通用户分不清，后续安装工具和文件权限容易混乱。

(2)看完这篇，你应该能做到

在控制台确认实例状态、公网 IP、地域、镜像和计费方式；检查安全组，只放行必要端口，SSH 的 22 端口优先限制来源 IP；用 Workbench 或本机 SSH 连接实例，确认登录用户、密钥文件权限和公网 IP；进入服务器后先更新系统，再确认 root / 普通用户权限边界。

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
