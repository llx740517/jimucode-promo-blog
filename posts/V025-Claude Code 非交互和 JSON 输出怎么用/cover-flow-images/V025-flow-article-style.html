Claude Code 非交互任务闭环：输入 → 校验

保留原流程：输入文件 → claude -p → 输出格式 → JSON 文件 → 校验 → 脚本 / 报告
STEP 1

准备输入

`bug-notes.md`、`requirements.md` 使用脱敏内容。

STEP 2

`claude -p`

打印结果后退出，只在可信目录里跑。

STEP 3

输出格式

text / json / stream-json，主线先看 json。

STEP 4

写入文件

`> output/review.json`，让脚本有稳定输入。

STEP 5

校验结构

`jq` 或 `python3 -m json.tool` 需实测确认。

STEP 6

接入脚本

报告、批量检查或 CI 前置检查再往后做。

失败排查顺序

终端输出输入文件配置文件Key/API 地址权益状态日志页网络权限项目目录

安全提示

`-p` 会跳过工作区信任弹窗；Key、Token、日志 ID、debug 日志和真实路径必须打码。

记忆线：输入文件 → 非交互命令 → JSON 文件 → 校验结构 → 接脚本 / 报告
