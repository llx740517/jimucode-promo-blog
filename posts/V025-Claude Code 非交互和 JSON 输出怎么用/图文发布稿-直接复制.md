# Claude Code 非交互怎么用：`-p`、JSON 输出和文件保存示范

这条用一个小型脱敏演示目录，录清楚 Claude Code 的非交互任务怎么跑：`claude -p`、`--output-format json`、`--json-schema`、输出到 `review.json`，再用 JSON 工具做校验。重点不是炫命令，而是把输入文件、输出文件、权限边界和失败排查顺序讲明白。

这篇适合刚开始接触Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按这条顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

(1)常见卡点

只会在终端里和 Claude Code 聊天，不知道怎样让它执行完就退出；以为 `--output-format json` 会直接得到自己想要的业务 JSON，忽略了 CLI 外层结果和业务结构的区别；自动化脚本里提示词写得太散，输出文件很难被下游脚本解析；录屏时容易只展示一条命令，看不出输入文件、输出文件和校验动作。

(2)看完这篇，你应该能做到

用 `claude -p "..."` 做一次非交互任务；理解 `text`、`json`、`stream-json` 三类输出更适合放在哪里；把一次任务重定向到 `output/report.json` 这类文件里；用 `--json-schema` 约束业务结果结构，并在录屏里解释为什么还要校验。

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
