macOS Codex 入门：Terminal、安装、Key、doctor、首次提问

STEP 1

Terminal / zsh

先确认 macOS、芯片架构和当前 shell。

echo $SHELL
STEP 2

基础环境

Node/npm 或 Homebrew 不必全有，但要知道走哪条路。

node -v / brew
STEP 3

安装 Codex

脚本、npm、Homebrew 选一条主线，不混跑。

codex-install.sh
STEP 4

配置 Key

Codex Key 全程打码，配置字段需实测确认。

sk-***
STEP 5

验证 doctor

先看命令位置、版本和本地诊断摘要。

codex doctor
STEP 6

首次提问

只在干净测试目录里验证，不直接改真实项目。

codex

通过标准

zsh 能找到 codex，版本和 doctor 可输出；测试目录能进入交互，日志页能核对请求状态。

常见错法

安装路线混用、Key 产品线拿错、权限提示没看清、旧 ~/.codex 配置未备份就清理。

总结Terminal/zsh → 环境 → 安装 → Key → doctor → 测试目录首次提问 → 日志。
