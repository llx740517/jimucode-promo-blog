# macOS 怎么安装 Codex：Terminal、zsh、Key 配置到第一次提问

这条专门讲 macOS 上安装 Codex CLI：先确认 Terminal、zsh、Node/npm 或 Homebrew，再从积木站 CLI 文档选择 Codex macOS / Linux 安装路线，配置 Codex Key，验证 `command -v codex`、`codex -V`、`codex doctor --summary`，最后在干净测试目录里完成第一次提问，并到日志页核对请求状态。

这篇适合刚开始接触Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按这条顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

(1)常见卡点

不知道从哪里打开 Terminal，也不知道当前 shell 是不是 zsh；不清楚应该用官方脚本、npm、Homebrew 还是积木站脚本安装；安装后 `codex -V` 或 `codex` 没反应，不知道是 PATH、安装目录还是终端没重开；不敢在终端粘贴带 Key 的命令，担心录屏泄露。

(2)看完这篇，你应该能做到

打开 Terminal，确认 macOS、zsh、Node/npm 或 Homebrew 环境；从积木站 `/docs/cli` 找到 Codex 的 macOS / Linux 安装和配置命令；用官方资料确认 Codex CLI 的 macOS 安装入口：官方脚本、npm、Homebrew 三种方式；主线演示选择一种安装方式，不在同一台机器上重复跑多种安装。

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
