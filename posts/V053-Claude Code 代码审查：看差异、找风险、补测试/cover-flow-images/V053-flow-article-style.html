V053 · 流程图 v2

Claude Code 代码审查闭环

STEP 1

查看差异

git status

git diff -- src tests

确认没有动配置和敏感文件。

STEP 2

限制权限

允许读代码、看 diff、跑测试。

拒绝 .env / secrets。

STEP 3

Claude Code 审查

/code-review

或审查当前 diff。

ultrareview 需实测。

STEP 4

找风险

边界值、错误处理。

硬编码、缺失测试。

STEP 5

只补测试

不重写模块。

只补失败点。

控制改动范围。

STEP 6

验证复查

npm test 需实测。

二次 git diff。

记录 PR 检查项。

$ git diff -- src tests
> /code-review
$ npm test # 需录屏前实测确认
$ git diff -- src tests
正常

风险变成测试，再用验证命令确认。

常见错法

只看总结，不检查 diff 是否扩大。

为什么这样做

1. 审查对象来自真实差异。

2. 权限边界先挡住敏感文件。

3. 结果用测试和二次 diff 收束。

看差异 → 找风险 → 补测试 → 跑验证 → 复查 diff；真实命令和权限以录屏前实测为准。
