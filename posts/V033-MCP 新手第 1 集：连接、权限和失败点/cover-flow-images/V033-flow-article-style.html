V033 / MCP 连接与排查路径

MCP 新手先看连接链路

本集边界 只做总览，不承诺 GitHub、数据库、浏览器等 MCP 都当场跑通。
STEP 1

本地工具 / 远程服务

stdio 本地进程，或 HTTP 远程 MCP server。

STEP 2

配置文件

Claude Code 看 `.mcp.json`、`~/.claude.json`；Codex 配置需实测。

STEP 3

MCP Server

server 暴露工具、资源和提示词给客户端。

STEP 4

权限 / 登录

Token、OAuth、scope、项目是否信任。

STEP 5

调用验证

`list`、`get`、`/mcp`、工具数量和状态。

STEP 6

常见失败点

命令路径、URL、Token、网络、权限、项目目录。

正常

先 list / get 核对，再 add 具体 server。

常见错法

把 MCP 当开关，失败后只会重复添加。

排查顺序

终端输出 / 配置文件 / Key/API / 日志页 / 网络 / 权限 / 项目目录

总结 MCP 是连接层：工具服务 -> 配置 -> server -> 授权 -> 调用 -> 排查。
