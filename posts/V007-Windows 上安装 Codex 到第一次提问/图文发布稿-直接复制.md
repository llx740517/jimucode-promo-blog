# Windows 安装 Codex 教程：PowerShell、npm、Key 配置到第一次提问

这条视频按真实录屏路线演示 Windows 上安装 Codex：先检查 Node.js 和 npm，再执行安装命令，配置 Codex Key，查看 `.codex` 配置文件，重开终端做版本和健康检查，最后在干净目录里完成第一次提问，并回到日志页核对记录。

这篇适合刚开始接触Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按这条顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

(1)常见卡点

没先检查 Node.js / npm，直接执行安装或配置命令；Codex、Claude Code、Gemini 的 Key 混用，导致 401/403；安装完成后不知道用 `codex --version`、`codex login status`、`codex doctor --summary` 验证；配置脚本写入了 `.codex`，但没有重开终端，也没有开新会话验证。

(2)看完这篇，你应该能做到

打开 PowerShell，检查 Node.js 22+ 和 npm；按积木代码助手 CLI 文档安装 Codex；使用 Codex Key 配置 `%USERPROFILE%\.codex\auth.json` 和 `%USERPROFILE%\.codex\config.toml`；重开终端后验证版本、登录状态和本机健康检查。

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
