V031 / Codex + Claude Code 权限安全流程

AI 编程工具不要一上来给满权限

使用原则 不是追求最快授权，而是让每一步都知道 AI 正在碰什么。
STEP 1

识别风险

`.env`、Key、删除、联网、权限命令先标出来。

STEP 2

只读 / 计划

Codex 用 read-only；Claude Code 用 plan。

STEP 3

读项目列计划

先说明会看哪些文件、可能改哪里。

STEP 4

小步放开

只允许一个小任务、一个目录或一个文件范围。

STEP 5

审批前停住

看工具、路径、参数和风险词。

STEP 6

diff / test 确认

用 `git diff`、`npm test` 和日志核对结果。

正常

先只读、列计划，再按小任务放开。

常见错法

直接满权限，审批提示也不看路径。

风险词

rm / curl / sudo / chmod / --force / .env / secrets

总结 先只读，列计划，小步放开，看审批，最后 diff + test。
