## 完整正文备用

这条用同一个演示项目对比 Codex 和 Claude Code 的权限安全开场：Codex 看 `--sandbox` 和 `--ask-for-approval`，Claude Code 看 `--permission-mode`、`--tools` 和权限设置。重点不是堆参数，而是形成一个顺序：先只读/计划，再小步放开，审批前看工具、路径和风险词，最后用 `git diff` 和测试确认。

这篇适合刚开始接触积木代码助手、Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按图里的顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

常见卡点：
一上来就给工具满权限，AI 可能直接读写不该碰的文件
看见审批提示时只想点同意，没有先看命令、路径、风险词
分不清 Codex 的 `--sandbox` / `--ask-for-approval` 和 Claude Code 的 `--permission-mode` / `--tools`
不知道 `.env`、Key、Token、日志、订单、用户信息应该怎样避开镜头和聊天记录

看完这篇，你应该能做到：
先用只读或计划姿态进入项目
让 AI 读项目、列计划、指出可能风险
只对一个小任务放开写入或工具调用
审批前看命令、工具、路径和风险词

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
