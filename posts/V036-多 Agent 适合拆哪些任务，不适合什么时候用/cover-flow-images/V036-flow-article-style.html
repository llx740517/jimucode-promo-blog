V036 / 多 Agent 判断流程

多 Agent 适合拆哪些任务

核心问题：能否独立调查、独立验证，最后由主线程汇总。
STEP 1

判断任务

能否独立调查、独立验证、最后汇总。

STEP 2

分角色

读代码、查问题、写测试、复核风险。

STEP 3

配入口

Claude Code 用 agents；Codex 入口需实测。

STEP 4

并行执行

各自输出结论，不同时改同一核心文件。

STEP 5

主线汇总

选一个最小改动点，人工做决策。

STEP 6

diff / test

看修改范围、跑测试、解释边界。

适合

并行调查、代码审查、测试设计、方案对比。

不适合

一行小改、需求不清、强耦合、敏感权限。

底部标签

读代码 / 查问题 / 写测试 / 复核 / .claude/agents / --agent / --agents / multi_agent / git diff / npm test

总结多 Agent 是组织复杂任务，不是替代判断；最后必须回到 diff 和测试。
