# Codex / Claude Code 项目规则模板怎么写：前端、后端、Python、全栈示例

这条用一个脱敏演示目录，把前端、后端、Python、全栈四类项目的规则模板录成一套可复用流程。Codex 重点看 `AGENTS.md`，Claude Code 重点看 `CLAUDE.md` 和 `.claude/rules/`。视频里会展示如何写具体规则、如何让 AI 复述规则、如何看 diff 和测试结果，以及哪些内容必须打码。

这篇适合刚开始接触Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按这条顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

(1)常见卡点

规则文件写得太泛，比如“写高质量代码”，AI 很难稳定执行；把所有规则堆在一个文件里，前端、后端、Python、测试、权限混在一起。

不知道 Codex 和 Claude Code 的规则入口不同：Codex 重点看 `AGENTS.md`，Claude Code 重点看 `CLAUDE.md` 和 `.claude/rules/`

录屏时只展示模板文本，看不出模板如何绑定项目目录、任务和验证命令。

(2)看完这篇，你应该能做到

按“项目类型 -> 规则文件 -> 任务 -> 验证”的顺序录模板课；给前端项目写出可展示的 UI、路由、状态、测试规则；给后端项目写出接口、鉴权、错误处理、迁移和测试规则；给 Python 项目写出环境、格式化、类型检查、测试和数据脱敏规则。

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
