# Codex / Claude Code 配置后不能用，先查终端、配置、Key 和日志

Codex 或 Claude Code 配置后不能用时，不要一上来就重装或反复改 Key。先按顺序查：终端输出、配置文件、Key/API 地址、权益状态、日志页、网络、权限、项目目录。本条用脱敏示意画面整理一条可复用的排查路线，真实录屏时以现场命令输出和日志记录为准。

这篇适合刚开始接触Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按这条顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

(1)常见卡点

看到 401、429、permission denied、读写失败、网络超时等报错后，不知道先看哪里；只盯着 Key，忽略了终端当前目录、配置文件、权益状态、日志页和权限模式；Codex 和 Claude Code 的命令、配置路径、权限开关不一样，容易混着查；日志、订单、用量和 Key 都带敏感信息，录屏时容易暴露。

(2)看完这篇，你应该能做到

按一个清晰顺序理解本条视频的核心操作路线；知道关键页面、终端输出或配置位置应该看哪里；知道哪些信息发布前需要脱敏，哪些内容需要以录屏现场为准。

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
