完整正文备用

这条录屏只讲一件事：新手怎么买第一台阿里云 ECS 服务器。按购买页顺序看计费方式、地域、实例规格、镜像、系统盘、公网 IP、带宽、安全组、登录凭证和配置摘要。适合后续准备在服务器上安装 Codex、Claude Code 或其他 AI 编程工具的用户。

不做固定配置推荐，不展示真实订单和支付信息。页面和价格以录屏时账号实际显示为准。

这篇适合刚开始接触积木代码助手、Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按图里的顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

常见卡点： 购买页字段太多，不知道哪些会影响后续 SSH、安装 Node.js、安装 Codex 或 Claude Code 容易只看价格，不看地域、镜像、系统盘、公网 IP、安全组和登录凭证 安全组规则、22 端口、root 用户、密钥对这些概念一上来很混 创建成功后不知道去哪里确认实例是否真的可用

看完这篇，你应该能做到： 从 ECS 购买入口进入购买页，并区分快速购买和自定义购买 按“计费方式、地域、规格、镜像、系统盘、公网 IP、带宽、安全组、登录凭证、配置摘要”的顺序检查购买项 知道哪些信息会影响后续 SSH 登录和 AI 编程工具安装 创建后去 ECS 控制台确认实例状态，知道要等到 Running / 运行中后再继续下一步

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
