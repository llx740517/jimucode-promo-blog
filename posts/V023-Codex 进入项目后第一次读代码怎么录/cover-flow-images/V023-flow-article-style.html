Codex 第一次读项目流程

先读清楚 -> 小步修改 -> 验证结果

保留本条关键流程：目录、Git、只读、计划、小改、diff、验证。
STEP 1

选项目目录

确认主工作目录，不要混多个项目。

--cd ~/demo
>
STEP 2

看 Git 基线

先知道旧改动，后面 diff 才清楚。

git status --short
>
STEP 3

只读项目

让 Codex 说明入口、命令、风险。

-s read-only
>
STEP 4

提小任务

先说计划，限制必要文件。

small task
plan first
>
STEP 5

看 diff

看文件数量和每处增删是否相关。

git diff
>
STEP 6

跑验证

按项目实际命令测试或构建。

npm test
npm run build
>
STEP 7

说清边界

通过只代表当前演示范围通过。

review scope

正常

先让 Codex 读项目地图，再给一个小任务。

风险

直接让它优化全项目，最后验收失控。

总结条

AI 的回答是导航，diff 和验证才是录屏里最该展示的证据。
