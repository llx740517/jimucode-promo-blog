V034 / MCP 场景判断流程

MCP 场景判断与调用验证

本条不先承诺第三方服务都能跑通，先分清 GitHub、数据库、浏览器、Playwright 的用途和风险。
STEP 1

外部服务

GitHub / DB / Browser / Playwright。先判断它解决什么问题。

STEP 2

MCP Server

HTTP 连接远程服务，stdio 启动本地进程。

STEP 3

CLI / 配置

Codex 与 Claude Code 分开写，参数不要混用。

STEP 4

AI 工具读取

工具看到能力列表，但列表存在不等于调用成功。

STEP 5

调用验证

`list`、`get`、`/mcp` 只验证配置和状态。

STEP 6

失败点

Token、权限、网络、包名、参数、项目目录。

正常

公开仓库、测试库、本地页面；先跑最短验证路径。

常见错法

把 MCP 当万能开关，直接接生产系统或登录态后台。

排查顺序

终端输出 / 配置文件 / Key/API 地址 / 权益状态 / 日志页 / 网络 / 权限 / 项目目录

总结MCP 是连接层：工具服务 -> MCP Server -> 配置 -> AI 工具 -> 调用验证 -> 排查。
