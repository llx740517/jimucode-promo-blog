## 完整正文备用

这条用双窗口对比 Codex 和 Claude Code 的三类配置：权限、MCP、记忆/规则文件。Codex 重点看 sandbox、approval、`codex mcp`、`AGENTS.md`；Claude Code 重点看 permission mode、allow/deny、`.claude/settings.json`、`.mcp.json`、`CLAUDE.md`。录屏使用脱敏示意配置，不展示真实 Token 或个人 settings。

这篇适合刚开始接触积木代码助手、Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按图里的顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

常见卡点：
把 Codex 的 `~/.codex/config.toml` 和 Claude Code 的 `.claude/settings.json` 混成一回事
把 Codex 的 `AGENTS.md` 和 Claude Code 的 `CLAUDE.md` 都叫“记忆”，但不知道加载范围不同
MCP 只知道“能连接工具”，不知道两边命令、配置文件、调试方式不一样
权限配置只看“能不能自动改”，没有区分审批、沙盒、工具 allow/deny、危险模式

看完这篇，你应该能做到：
用双窗口看清 Codex 和 Claude Code 在权限、MCP、记忆配置上的入口差异
知道 Codex 侧优先看 `codex --help`、`codex mcp --help`、`~/.codex/config.toml`、`AGENTS.md`
知道 Claude Code 侧优先看 `claude --help`、`claude mcp --help`、`.claude/settings.json`、`.mcp.json`、`CLAUDE.md`
遇到配置不生效时，先分清是权限问题、MCP 连接问题，还是记忆/规则文件没有被读到

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
