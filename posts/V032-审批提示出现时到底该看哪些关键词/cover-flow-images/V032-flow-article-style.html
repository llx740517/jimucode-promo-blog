V032 / 审批提示关键词检查路径

审批提示出现时，按这条路线看

核心原则 看不懂就先拒绝，改成更小任务，再重新申请权限。
STEP 1

复现审批提示

只在测试目录里触发，不拿真实项目冒险。

STEP 2

看工具 / 命令

Codex 看 shell；Claude Code 看工具名。

STEP 3

看目标路径

确认是不是当前工作区或测试目录。

STEP 4

看动作类型

读、写、删除、覆盖、推送要分开判断。

STEP 5

查敏感数据

`.env`、secrets、ssh、token、API key 单独看。

STEP 6

看网络 / 权限

curl、wget、chmod、chown、git push 先慢下来。

STEP 7

拒绝 / 改小 / 允许

批准后用 diff、测试和日志确认实际结果。

正常

先看关键词，确认边界，再允许。

常见错法

只看允许按钮，不看命令、路径和动作。

排障标签

终端输出 / 配置文件 / 权益状态 / 日志页 / 网络 / 权限

总结 工具或命令 -> 路径 -> 动作 -> 敏感数据 -> 网络权限 -> diff 验证。
