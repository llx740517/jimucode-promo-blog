Claude Code 第一次读项目：先计划 → 再修改

保留原流程：选目录 → 检查环境 → plan → 读项目 → 看权限 → 小改动 → 验证 → 复盘
STEP 1

选项目目录

不要在桌面、下载、整个 home 目录启动。

→
STEP 2

检查环境

看 version、help、Git 状态，确认项目干净。

→
STEP 3

计划模式

`--permission-mode plan`，先读不改。

→
STEP 4

读项目

README、package.json、src、项目规则。

STEP 5

权限核对

/status、/permissions；谨慎使用 --add-dir。

→
STEP 6

小改动

先确认计划，只改一处文案或一个文件。

→
STEP 7

验收

看 git diff，再跑 npm test / build / lint。

→
STEP 8

复盘边界

第一次小范围，后续再扩大任务。

底部检查标签

项目根目录git statusplan/status/permissionsgit difftest/build

常见错误

进错目录跳过权限不看 diff
记忆线：目录先确认，plan 先读不改，权限先看清，改完一定看 diff。
