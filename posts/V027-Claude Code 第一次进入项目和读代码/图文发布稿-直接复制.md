# Claude Code 第一次进入项目怎么用？先读代码、写计划、看 diff

这条讲 Claude Code 第一次进入项目的完整录屏路线：确认项目目录，使用 `--permission-mode plan` 先读项目和写计划，核对 `/permissions`、`/status`，谨慎使用 `--add-dir`，再做一个小改动，最后看 `git diff` 并运行测试或构建验证。演示使用脱敏测试项目，不展示真实 Key 或真实业务仓库。

这篇适合刚开始接触Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按这条顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

(1)常见卡点

进入项目后不知道先让 Claude Code 读哪些文件；容易一上来要求“直接修”，没有先看目录、规则、测试命令和风险；不理解 `--add-dir` 会扩大可访问目录，容易把无关目录也暴露给工具；不会看 Claude Code 的计划、权限提示、diff 和测试结果。

(2)看完这篇，你应该能做到

在干净演示项目里启动 Claude Code，并确认当前版本和帮助输出；用 `--permission-mode plan` 先让 Claude Code 读项目和写计划；知道 `--add-dir` 是授予额外目录访问，不是随便加目录；用 `/init` 生成或改进 `CLAUDE.md` 的起步说明，并知道真实项目里先审核再提交。

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
