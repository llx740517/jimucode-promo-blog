## 完整正文备用

这条用同一个脱敏演示项目，讲清 `AGENTS.md` 和 `CLAUDE.md` 的区别：Codex 侧怎么写项目规则，Claude Code 侧怎么写项目记忆，哪些内容适合放进去，哪些敏感信息不要写。视频会演示规则文件、终端验证、diff 和测试命令，适合刚开始做 AI 编程工作流的人。

这篇适合刚开始接触积木代码助手、Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按图里的顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

常见卡点：
不知道规则文件到底写项目说明、代码风格，还是写一大段提示词
把 API Key、账号信息、私有路径写进规则文件，后续容易泄露
在同一个项目里同时放 `AGENTS.md` 和 `CLAUDE.md`，但不知道哪个工具读哪个文件
规则写得太长，AI 反而抓不住重点

看完这篇，你应该能做到：
用一个演示项目拆出规则文件的 5 个模块：项目概览、常用命令、代码风格、修改边界、验收方式
分别说明 Codex 的 `AGENTS.md` 和 Claude Code 的 `CLAUDE.md` 录屏画面、命令差异、配置差异和适用场景
给出可录屏的小场景：让 AI 先读规则，再修一个小文案或测试失败点，最后看 diff 和验证命令
说明哪些内容不要写进规则文件，尤其是 Key、Token、订单、日志、账号和私有服务器信息

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
