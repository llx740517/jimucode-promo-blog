V003 Claude Code 排查流程

Key、地址、权限：先分层再处理

code.jimuxyz.com
STEP 1

终端输出

先看版本、认证状态和 doctor 输出。

claude auth status --text
STEP 2

配置文件

用户级、项目级、本机项目级 settings 都要看。

~/.claude/settings.json
STEP 3

Key / API

Key 和地址分开核对，页面写法需实测确认。

/docs/cli 实测确认
STEP 4

日志页

有记录看服务端状态；无记录回查本机和网络。

/logs
STEP 5

权限提示

权限拒绝不是 Key 失败，先看它要访问什么。

/permissions

正常判断

终端、配置、Key、日志、权限分层看；哪一层不一致，就回到那一层处理。

常见错法

把未登录、地址错误、权限拒绝、日志无记录全部混成“Key 不通”。

总结终端输出 → 配置文件 → Key/API → 权益状态 → 日志页 → 网络 → 权限 → 项目目录
