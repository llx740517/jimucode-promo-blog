完整正文备用

这条演示 Claude Code 配置失败时的排查顺序：先看终端输出和认证状态，再看 settings 配置、Key/API 地址、权益和日志页，最后再查网络、权限和项目目录。适合已经安装 Claude Code、但 Key 或地址配置后仍然不通的同学。录屏发布前默认重点检查二维码；Key、账号、用量、日志等内容是否遮挡，以本次剪辑任务的明确要求为准。

这篇适合刚开始接触积木代码助手、Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按图里的顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

常见卡点： 看到 401、403、authentication failed、permission prompt 等提示时，容易直接重装或换电脑 不知道 Claude Code 的登录状态、配置文件、环境变量和项目权限分别在哪里看 Key/API 地址写错、Key 过期、权益不足、日志页无记录、项目目录权限不足，这几类问题容易混在一起 配置里有 Key、Token、日志 ID、用户信息，录屏时容易泄露

看完这篇，你应该能做到： 用 claude --version、claude auth status --text、claude doctor 先确认 Claude Code 本机状态 区分“认证没登录”“Key/API 地址不对”“权限被拒绝”“日志/权益异常”“网络或项目目录问题” 知道要检查 ~/.claude/settings.json、.claude/settings.json、.claude/settings.local.json、~/.claude.json 这些 Claude Code 相关位置 知道 Key、Token、订单、日志、用户信息等画面必须提前打码

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
