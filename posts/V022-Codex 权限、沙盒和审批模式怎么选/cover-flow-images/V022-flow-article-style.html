Codex 权限选择流程

任务类型 -> 权限 -> 审批检查

保留本条关键流程：先按任务选权限，再核对状态、审批动作和 Git diff。
STEP 1

判断任务

只读项目、改当前项目、自动化、满权限。

task scope
>
STEP 2

选择沙盒

从窄到宽，不把满权限当默认。

read-only
workspace-write
danger-full-access
>
STEP 3

选择审批

真人边看边做，优先 on-request。

on-request
untrusted
never
>
STEP 4

核对状态

不要靠记忆，直接看当前会话。

/permissions
/status
/debug-config
>
STEP 5

批准前检查

看命令、路径、网络、敏感文件。

command / path
network / .env
>
STEP 6

运行后验收

看 Git 状态和 diff，确认改动范围。

git status
git diff

正常

先小权限试跑，改完用 `/status` 和 diff 核对。

风险

直接给满权限，或看见审批提示就允许。

总结条

审批不是按钮，是检查命令、路径、网络和敏感文件的机会。
