## 完整正文备用

这条录屏演示 ECS / Linux 服务器第一次登录后的基础安全检查：先看安全组和 22 端口，再 SSH 登录确认 root/普通用户，检查系统版本、端口、防火墙、磁盘、内存、Node/npm/PATH，最后确认 Git 状态和快照入口。适合在安装 Codex 或 Claude Code 前先打好服务器安全底座。

这篇适合刚开始接触积木代码助手、Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按图里的顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

常见卡点：
买完服务器就直接安装工具，没有先看安全组和开放端口
习惯用 root 操作，把 Key、配置、项目都放在 root 用户下面
不知道如何确认当前系统版本、磁盘、内存、进程、Node/npm 路径
不清楚运行 AI 编程工具前为什么先提交 Git、做快照

看完这篇，你应该能做到：
第一次 SSH 登录后先做一套服务器安全检查，而不是直接装 Codex 或 Claude Code
在 ECS 控制台确认实例状态、安全组、22 端口来源、快照入口和登录凭证
在 Linux 终端确认当前用户、系统版本、防火墙、开放端口、进程、磁盘、内存和 PATH
明白 root 和普通用户的差异，知道为什么建议用普通用户跑 AI 编程工具

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
