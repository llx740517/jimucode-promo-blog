V018 浅色图文流程

服务器首次登录安全检查

保留本条关键流程：安全组、SSH、用户、端口、防火墙、资源、PATH、Git、快照。
STEP 1

ECS / 安全组

确认实例、公网 IP、22 端口来源和快照入口。

STEP 2

SSH 登录

IP、用户名、密钥路径都用脱敏占位。

STEP 3

用户权限

whoami、id、HOME；区分 root 和普通用户。

STEP 4

系统更新

先看发行版，不把 apt、dnf、yum 混用。

STEP 8

快照 / 备份

云盘和快照入口需录屏前实测确认。

STEP 7

Git 状态

运行前看 git status，先知道改动基线。

STEP 6

资源 / PATH

df、free、node、npm、PATH 先检查。

STEP 5

端口 / 防火墙

ss、ufw、firewalld；先判断再调整。

正常先确认底座，再安装工具。
风险长期 root 和端口过大要谨慎。
验证资源、PATH、Git 都要看状态。
回退快照不能替代 Git。
检查标签

安全组 / 22 端口 / root / 普通用户 / 防火墙 / 磁盘 / 内存 / PATH / Git / 快照
