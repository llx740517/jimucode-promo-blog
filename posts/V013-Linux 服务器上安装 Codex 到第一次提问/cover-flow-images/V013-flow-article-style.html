V013 · Linux 服务器安装 Codex

服务器首次跑通 Codex：8 步流程 + 排查顺序

保留原流程关键步骤：ECS/安全组、SSH、普通用户、Node/Git、安装、配置、首次提问、日志核对。
article-style flow
STEP 1

ECS / 安全组

演示服务器、快照、必要端口；真实 ID 和 IP 不展示。

STEP 2

SSH 登录

连接测试服务器；用户名和密钥路径需遮挡。

STEP 3

普通用户

先跑 whoami、id、HOME，避免写进 /root。

STEP 4

Node / Git

确认 Node.js 22+、npm、Git；发行版命令需实测。

STEP 5

安装 Codex

环境检查、npm 安装、codex -V 和 help。

STEP 6

配置 Key

测试 Key 写入 ~/.codex；auth.json 不打开。

STEP 7

首次提问

干净目录，只读问题，不修改生产项目。

STEP 8

日志核对

/logs 看时间、产品线和状态；日志 ID 打码。

失败时先查什么

终端输出配置文件Key/API 地址权益状态日志页网络权限项目目录

正常 / 常见错法

正常

普通用户配置，doctor 通过后进入测试目录。

常见错法

root 和普通用户混用，或者把 Key 留在命令历史。

结论：服务器教程不是复制一条命令，而是把用户、环境、配置、首次提问和日志链路分清。
