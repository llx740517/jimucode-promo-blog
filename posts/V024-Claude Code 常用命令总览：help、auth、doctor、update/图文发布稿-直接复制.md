# Claude Code 常用命令总览：help、auth、doctor、update

这期只讲 Claude Code 入门最常用的 CLI 命令：`claude --help`、`claude --version`、`auth login/status/logout`、`doctor`、`update`、`install stable/latest` 和 `setup-token` 的边界。重点是先搞清命令顺序：先看版本和 help，再看登录状态，再用 doctor 查环境，最后再考虑更新或进入项目。账号、Token、路径和日志画面均需打码。

这篇适合刚开始接触Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按这条顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

(1)常见卡点

只知道输入 `claude`，但不知道 `claude --help` 里哪些命令最常用；不知道 `auth login`、`auth status`、`setup-token` 分别适合什么场景，容易在镜头里暴露账号或 Token；遇到命令找不到、登录状态不对、版本旧、更新失败时，不知道先看 `--help`、`--version` 还是 `doctor`；把 `--continue`、`--resume`、`--model`、`--add-dir` 这些选项混成“必须先学完”的高级内容，入门视频失焦。

(2)看完这篇，你应该能做到

先用 `claude --help` 和 `claude --version` 确认当前 Claude Code 是否可用；知道 `claude`、`claude "query"`、`claude -p "query"` 的区别：交互式、带初始提示、非交互输出；知道 `claude auth login`、`claude auth status --text`、`claude auth logout` 的用途，以及录屏时哪些账号信息要打码；知道 `claude doctor` 用来做本机健康检查，输出里可能包含路径、版本、安装方式，录屏要先看一遍。

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
