V035 / 三层判断流程

Skill、Agent、插件怎么区分

先问需求属于哪一层：复用做法、分派角色，还是打包给团队安装。
需求出现
STEP 1

Skill

重复方法 / 规则

写成可复用说明
沉淀团队清单
调用验证
STEP 2

Agent

专门角色 / 后台任务

限定工具和权限
配置 name / description
结果回收
STEP 3

插件

团队分发 / 安装更新

打包 skills / agents
管理命令或 MCP
启用验证

正常

先看版本、入口、文件位置，再做最小验证。

常见错法

把 Skill 当 Agent，把插件当 MCP。

排查顺序

命令版本 / 文件位置 / 权限范围 / 插件状态 / 日志或 debug

总结Skill 是方法，Agent 是角色，插件是包装层；入口和排查路径要分开。
