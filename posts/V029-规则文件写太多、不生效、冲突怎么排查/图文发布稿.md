## 完整正文备用

这条用一个脱敏演示项目，排查 Codex / Claude Code 规则文件不生效的问题：先看终端输出和当前目录，再查 `AGENTS.md`、`CLAUDE.md`、`CLAUDE.local.md` 是否加载，接着找规则过长、命令冲突、禁区冲突，最后再看 Key/API、权益、日志、网络和权限。适合已经写过项目规则，但 AI 仍然没按预期执行的人。

这篇适合刚开始接触积木代码助手、Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按图里的顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

常见卡点：
规则文件越写越长，最后 AI 只记住一部分，甚至抓错重点
同一个项目里有根目录规则、子目录规则、本地个人规则，互相说法不一致
当前终端不在项目根目录，工具读到的规则文件不是自己以为的那个
规则里写了旧路径、旧命令、旧测试方式，AI 反而被过期信息误导

看完这篇，你应该能做到：
给出一套排查顺序：先复现现象，再看终端输出、项目目录、规则文件、冲突内容、工具配置，最后再看 Key/API、权益、日志、网络和权限
分别说明 Codex 和 Claude Code 的录屏画面、命令差异、配置差异和适用场景
用一个小演示项目复现“规则写太多 / 规则冲突 / 当前目录不对”的问题，再把规则收敛到短而可验证
让观众知道哪些信息适合写进规则文件，哪些应该放到 settings、权限配置、日志排查或现场任务里

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
