Codex 新手从 0 到 1：先跑通再写代码

STEP 1

产品线

打开 /pricing，先确认当前配置的是 Codex。

/pricing
STEP 2

Codex Key

在 /keys 创建测试 Key，完整内容不进公开画面。

Key: sk-***
STEP 3

CLI 配置

按 Windows PowerShell 文档检查环境并写入配置。

codex version
STEP 4

首次运行

进入干净测试目录，让 Codex 先读取目录。

codex exec
STEP 5

日志核对

回到 /logs 看当天请求记录是否能对上。

/logs success

正常判断

终端能返回、日志能对上，说明网站 Key 到 CLI 请求这条线已经打通。

常见错法

混用 Key、旧终端不重开、直接进入真实项目，都会放大排查难度。

总结网站 → Key → CLI → 测试目录 → 日志核对；任何一步失败，就停在当前层排查。
