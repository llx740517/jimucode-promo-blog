Claude Code 权限路线：先保守 → 再放开

保留原流程：识别风险 → 选择权限模式 → 限制工具范围 → 写权限规则 → 看权限提示 → 验证结果
STEP 1

识别风险

.env、secrets、删除、联网、真实路径先标出来。

STEP 2

选择模式

新手从 plan / default 开始，不从高权限起步。

STEP 3

限制工具

`--tools Read,Grep,Glob` 才是限制工具范围。

STEP 4

写规则

allow / ask / deny 分开写，敏感读取先 deny。

STEP 5

看提示

停住看工具名、命令、路径、是否联网或删除。

STEP 6

验证结果

改完看 `git diff`；测试命令需录屏前实测确认。

底部检查标签

permission modeplandefault--tools--allowedTools--disallowedTools.claude/settings.json.env/permissionsgit diff

关键提醒

`allowedTools` 是减少提示；限制可用工具范围用 `--tools`。危险模式不作为日常默认工作流。

记忆线：先计划，再限制工具，敏感文件先 deny，改完一定看 diff
