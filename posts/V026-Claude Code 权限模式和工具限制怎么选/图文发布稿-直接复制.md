# Claude Code 权限模式怎么选：permission mode、tools、allowedTools 讲清楚

这条用一个脱敏演示目录讲 Claude Code 权限怎么选：什么时候用 `plan`，什么时候用 `--tools` 限制读和搜索，`--allowedTools` 和 `--disallowedTools` 分别解决什么问题，`.claude/settings.json` 里的 allow / ask / deny 怎么理解。重点是让新手先保守进入项目，看清权限提示，再用 diff 和测试确认结果。

这篇适合刚开始接触Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按这条顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

(1)常见卡点

不知道 `plan`、`default`、`acceptEdits`、`auto` 和 `bypassPermissions` 什么时候用；看到权限提示就直接确认，没有看清工具名、命令和路径；把 `--allowedTools` 误认为工具白名单，忽略了官方资料里“限制工具范围应使用 `--tools`”的区别；项目里有 `.env`、secrets、Token，但没有写 `permissions.deny`。

(2)看完这篇，你应该能做到

按一个清晰顺序理解本条视频的核心操作路线；知道关键页面、终端输出或配置位置应该看哪里；知道哪些信息发布前需要脱敏，哪些内容需要以录屏现场为准。

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
