## 完整正文备用

准备在 Windows 上装 Codex 或 Claude Code 之前，先别急着复制安装命令。本条用 PowerShell 检查 Node.js、npm、Git、VS Code CLI，并准备一个干净测试目录。后面再安装时，如果出错，就能更快判断是环境、PATH、工具安装、Key 还是日志问题。

这篇适合刚开始接触积木代码助手、Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按图里的顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

常见卡点：
直接复制安装命令，结果 `npm`、`git` 或 `code` 命令找不到
PowerShell 和 CMD 混用，看到提示符不同就不知道自己在哪个终端里
VS Code 已安装，但 `code .` 无法从终端打开当前文件夹
Git for Windows、Git Bash、PowerShell 的作用混在一起，尤其影响 Claude Code 的 shell 行为判断

看完这篇，你应该能做到：
打开 PowerShell，确认当前不是 CMD
用 `node -v`、`npm -v` 检查 Node.js 和 npm
用 `git --version` 检查 Git，并知道 Git for Windows 对 Claude Code 的 Git Bash 价值
用 `code --version` 和 `code .` 检查 VS Code 命令行入口

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
