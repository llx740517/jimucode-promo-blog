## 完整正文备用

这条专门讲 Windows 本地排查：`codex`、`claude`、`node`、`npm`、`git`、`code` 命令找不到时，先看终端输出，再用 `where.exe` 查命令位置，看当前 PATH、执行策略、配置文件、Key/API 地址、权益状态和日志页。Codex 和 Claude Code 分开判断，不把安装问题、Key 问题和网络问题混在一起。

这篇适合刚开始接触积木代码助手、Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按图里的顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

常见卡点：
`codex`、`claude`、`node`、`npm`、`git`、`code` 其中某个命令找不到
装完软件没有重开 PowerShell，PATH 仍然是旧的
PowerShell 和 CMD 混用，复制了不适合当前终端的命令
执行 `.ps1` 脚本时被 ExecutionPolicy 拦住

看完这篇，你应该能做到：
先看终端输出，确认是“命令找不到”、执行策略、乱码，还是工具自身报错
用 `where.exe` 查命令到底从哪里被找到
看当前 PowerShell 的 PATH，而不是直接修改系统变量
用 `Get-ExecutionPolicy -List` 判断是否是脚本执行策略问题

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
