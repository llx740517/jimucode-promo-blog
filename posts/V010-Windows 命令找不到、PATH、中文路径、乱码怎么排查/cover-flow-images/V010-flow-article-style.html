V010 Windows 命令找不到排查
code.jimuxyz.com

Windows 报错先分层：命令、PATH、策略、路径、日志

STEP 1

终端输出

先看错误原文，分清命令找不到、脚本被拦或认证失败。

codex --version
STEP 2

命令位置

用 where.exe 看命令是否被当前 PATH 找到。

where.exe codex
STEP 3

PATH

先看当前会话，安装后通常要重开 PowerShell。

$env:Path
STEP 4

执行策略

.ps1 被拦时查策略，不当作普通命令错误。

Get-ExecutionPolicy
STEP 5

英文目录

换干净英文路径，排除中文、空格、乱码干扰。

jimu-v010-demo
STEP 6

分线查看

Codex 与 Claude Code 的命令、Key、日志不要混用。

/docs/cli

通过标准

命令可定位，PATH 可解释，脚本策略清楚，干净目录可复现；工具能启动后再查 Key、权益和日志页。

常见错法

PowerShell / CMD 混用、没重开终端、中文路径里排查、Codex 和 Claude Code 产品线混用。

总结终端输出 → where.exe → PATH → 执行策略 → 英文目录 → 文档 / Key / 日志。
