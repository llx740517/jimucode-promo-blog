## 完整正文备用

这条录屏只讲 Codex CLI 的基础命令顺序：先用 `codex --help` 看入口，再用 `codex login status` 确认登录，用 `codex doctor` 检查本地环境，最后看 `codex update` 的更新入口。适合刚安装 Codex、准备进入项目之前的新手。

本条不做项目实战，不混讲 Claude Code，不展示真实 Key。命令输出以录屏当天的 Codex CLI 版本为准。

这篇适合刚开始接触积木代码助手、Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按图里的顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

常见卡点：
不知道安装后第一条命令应该跑 `codex --help` 还是直接 `codex`
不知道 `login`、`logout`、`login status` 分别用来确认什么
本地环境出问题时只会重装，不知道先跑 `codex doctor`
不清楚 `update` 什么时候录、什么时候不该现场升级

看完这篇，你应该能做到：
用 `codex --help` 看当前 Codex CLI 可用命令
用 `codex login` 或 `codex login status` 确认登录状态
知道 API Key 登录可以通过 `codex login --with-api-key` 从标准输入读取 Key，但录屏中不能展示真实 Key
用 `codex doctor --summary --ascii --no-color` 做一版适合录屏的本地诊断输出

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
