## 完整正文备用

这条视频按真实录屏路线演示 Claude Code 新手第一步：先在积木代码助手确认 Claude Code 产品线和 Key，再按 Windows PowerShell 检查 Node、npm、Git，配置环境变量，确认登录/鉴权状态，最后进入一个干净项目目录做首次运行，并回到用量日志核对记录。

这篇适合刚开始接触积木代码助手、Codex 或 Claude Code 的同学。不要只盯着一个按钮或一条命令，建议按图里的顺序看：先看当前问题，再看操作路径，最后确认结果有没有真正跑通。

常见卡点：
把 Codex Key、Claude Code Key、Gemini Key 混用，导致 401/403 后反复重装
只安装了 Claude Code，但没有确认 `ANTHROPIC_BASE_URL`、`ANTHROPIC_API_KEY` 等环境变量是否生效
直接在真实项目里运行 Claude Code，遇到权限提示、Git Bash 缺失、项目目录不对时不知道先查哪里
不知道 `claude auth status --text`、`claude doctor`、`claude -p` 和交互式 `claude` 分别适合什么验证场景

看完这篇，你应该能做到：
在积木代码助手网站确认 Claude Code 产品线、Key 管理和 CLI 文档入口
在 Windows PowerShell 中检查 Node.js、npm、Git / Git Bash
安装或确认 Claude Code 可运行，并用 `claude --version` 验证
配置 Claude Code Key 对应的环境变量，重开终端后确认登录/鉴权状态

我的建议是，第一次操作时不要一边改很多地方，一边猜原因。先把页面、终端输出、配置文件、日志记录这几块分开看，哪一步不一致，就从那一步往回查。

如果你也在配置或使用 AI 编程工具，可以先收藏这篇。后面遇到类似问题时，按这条路线重新核对一遍，通常能更快判断下一步该看哪里。
